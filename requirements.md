## FUNCTIONAL REQUIREMENTS 

1. The system lets the players create and manage a party made up of 2 to 4 characters.
2. The game generates and manages foes during exploration and combat.
3. The system lets the players combat Destruction's minions and advance in the story.
4. The system lets the players make plot choices which influence the ending.
5. The system saves and loads the players' progress.
6. The game updates the game state based on Destruction's progress.
7. The system manages dialogues, players' inventories and quests.
8. The system manages boss fights against Destruction and its emanations.
9. The system allows multiple endings based on the players' decisions.

## NON FUNCTIONAL REQUIREMENTS 

1. The game should run at 30 FPS in its main areas.
2. Loading times should be below 10 seconds.
3. The game should run on both Windows and Linux.
4. Saves should be resistant to corruption.
5. The user interface should be simple to browse in few steps.
6. The code should be modular to make patches and expansions easy.
7. The game should not crash during saves or loads.
8. The lore should be coherent with the Creation-Destruction cycle.
9. The game should visibly show the advancement of Destruction.
